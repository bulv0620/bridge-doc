# 📁 文件同步

## 功能截图

![File Sync](http://bucket.bulv.cc/bridge/file-sync.png)



## 同步会话

Bridge允许创建多个同步会话，且支持多会话并行处理（过多的并行任务可能会造成资源占用过高和软件卡顿）。

点击`+`图标创建一个新会话，点击`X`关闭会话，双击会话Tab可以修改会话的名称。

创建的会话会被缓存，应用重启后会自动恢复，无需每次重复填写配置。



## 同步配置信息

你只需要选择同步源、同步目标，并确定同步策略，即可进行比对和同步。

支持多种同步源

- **本地目录**：
  - 选择本地路径即可
- **FTP**：
  - 填写连接参数： `Host(主机)`、`Port(端口)`、`Username(用户名)`、`password(密码)`
  - 选择路径
- **S3**：
  - 填写连接参数： `Region(区域)`、`Bucket Name(桶名称)`、`Access Key ID(访问密钥)`、`Secret Access Key(密钥)`、`Custom Endpoint(自定义Endpoint)`
  - 选择路径

支持多种同步策略

- **镜像同步**：目标目录完全复制源目录结构
- **增量同步**：覆盖相同文件、复制新增文件、保留目标端旧文件
- **双向同步**：双向复制更新，冲突文件默认忽略（可在比对结果树中手动处理）



## 同步操作栏

操作栏放置了一系列按钮并做好了防呆，根据当前的状态程序会自动禁用、启用相应的操作按钮，以降低上手难度以及避免一些意料之外的错误。

按钮有以下这些：

- `Compare(对比)`：当你选择完同步源和同步目标后，即可开始比对操作；
- `Stop(结束)`：在比对操作进行时，可以随时结束，并获取当前已经比对出来的结果；
- `Sync(同步)`：当比对任务进行后，并获取到了比对结果即可进行同步操作；
- `Pause(暂停)`：在同步操作进行时，你可以随时暂停，暂停后再次点击同步即可恢复。

在同步栏的右侧还有一个按钮：`Ignored Folders(忽略文件夹)`

点击该按钮会弹出忽略文件夹维护窗口，你可以维护比对时需要忽略的文件夹，确认后再次进行比对将忽略文件树中同名的所有文件夹。



## 比对结果树

比对结果将以文件树表格的形式展示，单行中会展示源信息和目标信息，并以颜色标注直观的区分对该差异执行的操作：

- **绿色标注**：最终保留的文件
- **红色标注**：将被删除的文件
- **灰色标注**：忽略掉文件

在比对完成后，程序会自动根据三种同步策略来决定源/目标文件的去留，不过你仍然可以自由的决定每个文件的走向。

点击`Resolution(操作)`列中的⬅️按钮、➡️按钮即可直观、快速的决定文件走向。



## 同步状态信息

页面底部提供了当前会话的同步状态信息

- 已传输大小 / 总传输大小
- 已传输文件数 / 总传输文件数
- 进度条
- 百分比

当比对完成后会展示总传输大小和总传输的文件数，并在同步进行时实时更新已传输进度
